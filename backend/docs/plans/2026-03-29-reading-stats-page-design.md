# 阅读统计页面重新设计

## 概述

重新设计阅读统计页面，提供详细的阅读信息和记录展示，包括阅读趋势图表、时段分布分析和文章阅读进度详情。

**日期：** 2026-03-29
**状态：** 设计已批准

---

## 设计方案

### 布局方式

采用**仪表盘式布局**，所有内容在一个页面垂直展示，信息密度高，适合快速浏览。

```
┌─────────────────────────────────────┐
│  阅读统计                            │
├─────────────────────────────────────┤
│  📊 概览卡片（一行3个）               │
│  [总阅读时长] [已读文章数] [本周阅读] │
├─────────────────────────────────────┤
│  📈 阅读趋势图表（7天/30天切换）      │
│  [折线图：每日阅读时长 + 文章数]      │
├─────────────────────────────────────┤
│  ⏰ 阅读时段分布                     │
│  [饼图：时段分布] [热力图：活跃度]    │
├─────────────────────────────────────┤
│  📚 文章阅读进度详情                  │
│  [列表：带进度条的文章卡片]           │
└─────────────────────────────────────┘
```

## 功能模块

### 1. 概览卡片区域

在页面顶部显示3个关键指标卡片。

| 卡片 | 显示内容 | 数据来源 |
|------|---------|---------|
| 📖 总阅读时长 | 累计阅读小时数 | `ReadingStats.total_duration` 求和 |
| 📚 已读文章数 | 至少读过一次的文章数 | `ReadingStats` 去重计数 |
| 🔥 本周阅读 | 本周累计阅读分钟数 | `ReadingHistory.started_at` 按周筛选 |

**视觉设计：**
- 一行3个卡片，响应式布局（移动端自动换行）
- 每个卡片包含：图标、数值、单位、同比变化
- 使用渐变背景区分卡片类型
- 数值使用等宽字体，确保对齐

---

### 2. 阅读趋势图表

显示阅读活动的趋势变化。

**图表类型：** 双轴折线图（或组合图）

**X轴：** 日期（可切换 7天 / 30天 / 90天）

**左Y轴：** 阅读时长（分钟）

**右Y轴：** 阅读文章数

**数据系列：**
- 📈 折线1：每日阅读时长（填充区域，渐变色）
- 📊 柱状图：每日阅读文章数

**交互功能：**
- 时间范围切换器（7天/30天/90天）
- 鼠标悬停显示详细数据
- 点击数据点跳转到对应日期的阅读历史

**数据聚合：**
```sql
SELECT DATE(started_at) as date,
       SUM(reading_duration)/60 as minutes,
       COUNT(DISTINCT article_id) as articles
FROM reading_history
WHERE started_at >= [date_range]
GROUP BY DATE(started_at)
ORDER BY date
```

### 3. 阅读时段分布

分析阅读时间的分布规律。

**子部分 A：时段分布饼图**

将一天划分为4个时段：
- 🌅 早晨 (06:00-12:00)
- ☀️ 下午 (12:00-18:00)
- 🌙 晚上 (18:00-24:00)
- 🌌 深夜 (00:00-06:00)

显示每个时段的阅读次数和时长占比。

**子部分 B：阅读热力图**

类似 GitHub contribution graph 的热力图：
- X轴：一周7天
- Y轴：一天24小时
- 颜色深浅表示阅读活跃度（次数）
- 鼠标悬停显示具体数据

**布局：** 两个图表并排显示（移动端上下堆叠）

**数据来源：** 从 `ReadingHistory.started_at` 提取小时和星期几

---

### 4. 文章阅读进度详情

展示每篇文章的详细阅读情况。

**列表项布局：**
```
┌────────────────────────────────────────────┐
│ 📕 [文章标题]                    [最后阅读] │
│ 进度: ████████░░░░ 75%  |  阅读次数: 3次  │
│ 累计时长: 45分钟  |  上次读到: 第3章     │
└────────────────────────────────────────────┘
```

**每篇文章显示：**
- 文章标题和标签
- 可视化进度条（0-100%）
- 阅读次数
- 累计阅读时长
- 最后阅读时间
- 最后阅读位置（如果有）

**交互功能：**
- 点击标题跳转到文章详情
- 按进度/时长/时间排序
- 搜索筛选文章

**样式设计：**
- 进度条使用渐变色
- 未读文章灰色显示
- 已读完文章（100%）绿色高亮

## 技术实现

### 后端新增接口

**1. 阅读趋势数据**
```
GET /api/v1/reading/trends?days=7
Response: {
  items: [{ date: "2026-03-29", minutes: 45, articles: 3 }]
}
```

**2. 时段分布数据**
```
GET /api/v1/reading/time-distribution
Response: {
  periods: [{ name: "早晨", count: 12, duration: 360 }],
  heatmap: [{ hour: 9, day: 1, count: 5 }]
}
```

**3. 文章进度详情**
```
GET /api/v1/reading/progress?page=1&size=20
Response: {
  items: [{
    article_id: 1,
    article_title: "...",
    total_views: 3,
    total_duration: 2700,
    reading_progress: 75,
    last_read_at: "2026-03-29T10:00:00Z"
  }]
}
```

### 前端技术栈

- **图表库：** Recharts（React 生态，轻量级）
- **样式：** Tailwind CSS
- **状态管理：** React hooks (useState, useEffect)

### 数据模型变更

无需新增数据模型，使用现有的 `ReadingHistory` 和 `ReadingStats`。
